# English Version

## General Explanations

### About the dataset
This dataset is used to predict the **progression of diabetes** one year after the initial patient data collection. Therefore, it is a regression problem.

#### Dataset structure:
- Contains 442 samples of patients with diabetes.
- Each sample has 10 numerical attributes: age, sex, body mass index (BMI), average blood pressure, and six related blood measurements such as total cholesterol levels, lipoproteins (LDL and HDL), cholesterol/HDL ratio, triglycerides, and glucose.

The target variable represents a quantitative measure of disease progression.

For more details about the dataset, access:
https://scikit-learn.org/stable/modules/generated/sklearn.datasets.load_diabetes.html

### About how the code was structured
The code was **designed to didactically complement** the **previous tutorial** ([Classification Tree for Breast Cancer](1.decision_tree_classification.md)). In it, we evaluated four approaches:
- Without pruning
- With pre-pruning
- With post-pruning
- With combined pre- and post-pruning

Additionally, we presented tree visualizations, confusion matrices, depth, number of nodes, and feature importance.

**In this regression tutorial, we could follow analogous structures and visualizations. However, to vary the approach and enrich learning, we chose to modify the analysis format:**

We used **three strategies:**
- **Without pruning**
- **With pre-pruning**
- **With post-pruning**

> We did not apply combined pre- and post-pruning to simplify our analysis, since in many cases there are no significant gains.

For each strategy, we have:
- **MSE and R²**
- **Actual vs. predicted values** to visually evaluate prediction quality.
- **Trade-off between tree complexity and error** (MSE), **varying** the parameter **ccp_alpha**.

## Code

### Import libraries

```python
import pandas as pd
import numpy as np
from sklearn.tree import DecisionTreeRegressor, plot_tree
from sklearn.datasets import load_diabetes
from sklearn.model_selection import train_test_split
from sklearn.metrics import mean_squared_error, r2_score
import matplotlib.pyplot as plt
import graphviz
```

- pandas and numpy: libraries for data manipulation.
- DecisionTreeRegressor and plot_tree: to create and visualize a regression tree.
- load_diabetes: example dataset from the scikit-learn library.
- train_test_split: to split data into training and testing sets.
- mean_squared_error, r2_score: metrics to evaluate the model.
- matplotlib.pyplot: for graphical visualizations.
- graphviz: to visualize the regression tree.

### Load and prepare the data

```python
data = load_diabetes()
X, y = data.data, data.target
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.3, random_state=123
)
```

- We load the Diabetes dataset into the variable `data`.
- `X` receives the features (10 numerical attributes related to medical exams).
- `y` receives the continuous target variable (a quantitative measure of disease progression).
- We split the data into training and test sets:
  - 70% of the data goes to training (X_train, y_train)
  - 30% goes to testing (X_test, y_test)
  - `random_state=42` ensures reproducibility (always the same split).

### Without pruning

#### Train the tree

```python
reg_none = DecisionTreeRegressor(random_state=123)
```

#### Evaluate the model

```python
y_pred_none = reg_none.predict(X_test)
```

> [!NOTE]

In this graph, we observe a large **dispersion of points**, suggesting predictions with **high variability**. This indicates **possible overfitting**: the model fits the training data too closely but **loses generalization ability**.

> With pre-pruning (max_depth=3), the tree produces **predictions at only a few fixed levels**, showing a **highly simplified model**. The predicted values concentrate in horizontal bands, far from the actual values, suggesting **underfitting**: the model **does not have enough capacity to capture data variability**.

### Post-pruning

#### Find alpha

```python
path = reg_none.cost_complexity_pruning_path(X_train, y_train)
ccp_alphas, impurities = path.ccp_alphas, path.impurities
```

#### Train one tree for each alpha value

```python
regs = []
for ccp_alpha in ccp_alphas:
    reg = DecisionTreeRegressor(random_state=123, ccp_alpha=ccp_alpha)
    reg.fit(X_train, y_train)
    regs.append(reg)
```

#### Evaluate training and testing error for each tree

```python
train_errors = [
    mean_squared_error(y_train, reg.predict(X_train))
    for reg in regs
]

test_errors = [
    mean_squared_error(y_test, reg.predict(X_test))
    for reg in regs
]
```

#### Plot error vs alpha to visualize trade-off

```python
plt.figure(figsize=(8,5))
plt.plot(ccp_alphas, train_errors, marker='o', label='Training error')
plt.plot(ccp_alphas, test_errors, marker='o', label='Test error')
plt.xlabel('Alpha')
plt.ylabel('Mean Squared Error (MSE)')
plt.title('Trade-off between tree complexity and error')
plt.legend()
plt.show()
```

![Trade-off](../.././figures/reg_tradeoff_pt.png)

> [!NOTE]
> The purpose of generating this plot is to visualize that there is an alpha value that minimizes the test error. The analysis validates the post-pruning process, showing that there is an optimal complexity point that balances bias and variance.

#### Select best alpha based on lowest test error

```python
best_alpha = ccp_alphas[np.argmin(test_errors)]
print(f"Best alpha: {best_alpha:.5f}")
```

**Output:**

Best alpha: 776.02184

#### Train final tree with best alpha

```python
reg_poda = DecisionTreeRegressor(random_state=123, ccp_alpha=best_alpha)
reg_poda.fit(X_train, y_train)
```

#### Model visualization

```python
plt.figure(figsize=(20,10))
plot_tree(
    reg_poda,
    filled=True,
    feature_names=data.feature_names,
    rounded=True,
    fontsize=12
)
plt.show()
```

**Output:**
![reg_poda](../../figures/reg_poda.png)

#### Evaluate pruned model

```python
y_pred_poda = reg_poda.predict(X_test)
mse_poda = mean_squared_error(y_test, y_pred_poda)
r2_poda = r2_score(y_test, y_pred_poda)

print(f"MSE (post-pruning): {mse_poda:.3f}")
print(f"R² (post-pruning): {r2_poda:.3f}")

# Actual vs Predicted plot - pruned
plt.figure(figsize=(6,6))
plt.scatter(y_test, y_pred_poda, alpha=0.6)
plt.plot([y_test.min(), y_test.max()], [y_test.min(), y_test.max()], 'r--')
plt.xlabel("Actual Values")
plt.ylabel("Predicted Values")
plt.title("Actual vs Predicted Values - Post-pruning")
plt.show()
```

**Output:**

MSE (post-pruning): 4049.847  
R² (post-pruning): 0.319  

![Actual vs Predicted - Post-pruning](../.././figures/reg_pos_pt.png)

> [!NOTE]
> With post-pruning, the simplification of the model becomes even more extreme. The predictions consolidate into an even smaller number of horizontal bands, characterizing a case of underfitting even more severe than pre-pruning.
