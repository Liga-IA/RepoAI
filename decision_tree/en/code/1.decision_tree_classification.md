## General Explanations

### About the dataset
This dataset is used to classify **breast tumors** as **malignant (cancerous)** or **benign (non-cancerous)** based on features extracted from medical imaging exams. Therefore, it is a binary classification problem.

#### Dataset structure:
- Contains 569 samples, with 212 malignant and 357 benign.
- 30 numerical attributes are analyzed, related for example to tumor characteristics such as radius, texture, perimeter, area, smoothness, etc.

For more details about the dataset, access:  
https://scikit-learn.org/stable/modules/generated/sklearn.datasets.load_breast_cancer.html

### About how the code was structured
The code was organized to compare the performance of different pruning strategies in decision trees.

The four evaluated approaches were:

**1. Without pruning** — the **tree grows freely** until it completely fits the training data.

**2. Pre-pruning** — **limits the depth** of the tree **during training** using the parameter **max_depth**.

**3. Post-pruning** — uses the parameter **ccp_alpha** to **simplify** the tree **after training**.

**4. Pre-pruning and Post-pruning** — applies **max_depth and ccp_alpha together**.

For each strategy, we have:
- Tree visualization.
- Training and testing accuracy.
- Tree depth.
- Total number of nodes.
- Confusion matrices.
- Feature importance.

### Load and prepare the data
```python
data = load_breast_cancer()
X, y = data.data, data.target
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.3, random_state=42, stratify=y
)
```

- Loads the Breast Cancer Wisconsin dataset into the variable `data`.
- `X` receives the features (30 numerical attributes).
- `y` receives the labels/classes (benign or malignant).
- Splits the data into training and testing sets:
  - 70% of the data goes to training (X_train, y_train)
  - 30% goes to testing (X_test, y_test)
  - `random_state=42` ensures reproducibility (always the same split).
  - `stratify=y` maintains the original class proportion (malignant/benign) in both sets.

> [!CAUTION]
> The `stratify` parameter can only be used when the target variable (y) is categorical (classification). In regression problems, where y is continuous, stratification does not make sense, which is why we do not use it in those cases.

### Training the model
In this case, we will do this 4 times for didactic purposes: without pruning, with pre-pruning, with post-pruning, and with both pre- and post-pruning.

#### 1. Without pruning
```python
clf_sem_poda = DecisionTreeClassifier(random_state=42)
clf_sem_poda.fit(X_train, y_train)
```

- The variable `clf_sem_poda` is the classifier that will be trained.
- `random_state=42` ensures reproducible results.
- We train the model using the training data (X_train and y_train).

#### 2. With pre-pruning
```python
clf_pre_poda = DecisionTreeClassifier(max_depth=4, random_state=42)
clf_pre_poda.fit(X_train, y_train)
```

> [!NOTE]
> Analogous to "Without pruning", with the difference that we use `max_depth=4`. This limits the tree growth, performing pre-pruning since the restriction is applied during training.

#### 3. With post-pruning
Post-pruning removes parts of the tree that do not contribute significantly to model improvement, helping to avoid overfitting.

The parameter **ccp_alpha (Cost Complexity Pruning Alpha) controls the pruning level:**
- **ccp_alpha = 0: no pruning is performed (full tree).**
- **ccp_alpha > 0: the higher the value, the more branches are cut.**

##### 3.1 Finding possible "alphas" and testing which provides better test accuracy
```python
path = clf_sem_poda.cost_complexity_pruning_path(X_train, y_train)
ccp_alphas = path.ccp_alphas
```

```python
best_alpha = 0
max_acc = 0

for alpha in ccp_alphas:
    # Ignore the last alpha that removes all nodes
    if alpha >= path.ccp_alphas[-2]:
        continue
    
    temp_clf = DecisionTreeClassifier(random_state=42, ccp_alpha=alpha)
    temp_clf.fit(X_train, y_train)
    acc = accuracy_score(y_test, temp_clf.predict(X_test))
    
    if acc > max_acc:
        max_acc = acc
        best_alpha = alpha
```

- We test each alpha to see which provides better performance on the test set.

##### 3.2 Creating the final post-pruned model with the best alpha
```python
clf_pos_poda = DecisionTreeClassifier(random_state=42, ccp_alpha=best_alpha)
clf_pos_poda.fit(X_train, y_train)
```

> [!NOTE]
> Analogous to the "Without pruning" case, but adding the parameter `ccp_alpha=best_alpha`, which applies post-pruning with the best found value.

#### 4. With pre-pruning and post-pruning
```python
clf_pre_pos_poda = DecisionTreeClassifier(random_state=42, max_depth=4, ccp_alpha=best_alpha)
clf_pre_pos_poda.fit(X_train, y_train)
```

> [!NOTE]
> We use the same parameters (`max_depth` and `ccp_alpha`) from items 2 and 3 together.
