## Typical Model Problems
Gradient Boosting is an ensemble learning algorithm that combines weak models, correcting errors and improving results sequentially, which makes it extremely powerful. However, like any technique, it has its challenges:

### 1. Overfitting
Overfitting occurs when the model becomes too specialized to the training data. Instead of learning the underlying patterns and true relationships in the data, it "memorizes" the training set. Consequently, it is unable to generalize, resulting in low accuracy when presented with test data.

As the model continues to train (adding more trees and complexity), its performance metric on the training set continues to improve or stays perfect. Meanwhile, in validation, the model fails to capture the broad and true relationships of the data, indicating a loss of generalization.

### 1.1 How to solve this problem?
The developer needs to monitor the model's performance during training and implement **early stopping** techniques. Furthermore, using cross-validation strategies, such as **k-fold cross-validation**, can provide more reliable estimates of model performance and hyperparameter tuning, further enhancing the predictive capabilities of gradient boosting.

## 2. Class Imbalance
The model generally seeks to minimize the total loss. To reduce overall errors, it may default to predicting the majority class. However, it then fails to identify instances of the minority class, becoming entirely biased in favor of the majority class.

### 2.2 Solutions
Class imbalance can be addressed using several techniques. A pre-processing approach is **oversampling** the minority class—for example, **SMOTE**, which generates new examples by interpolating between nearby minority samples. Another option is **undersampling** the majority class, reducing its number of examples. Alternatively (or in conjunction), one can use **weighted loss functions** that assign higher penalties for errors in the minority class, forcing the model to pay more attention to those instances. Ensemble-based techniques (such as BalancedBagging or EasyEnsemble) and adjusting the classifier's **decision threshold** are also useful.

```python
from imblearn.over_sampling import SMOTE
import matplotlib.pyplot as plt
import seaborn as sns

# Applying SMOTE
smote = SMOTE()
X_smote, y_smote = smote.fit_resample(original_data.drop(columns=['target']), original_data['target'])

fig, ax = plt.subplots(nrows=1, ncols=2, figsize=(10, 6))

# Plotting original data
sns.scatterplot(x='A', y='B', hue='target', data=original_data, ax=ax[0])
ax[0].set_title('Without SMOTE')

# Plotting data after SMOTE
sns.scatterplot(x='A', y='B', hue=y_smote, data=X_smote, ax=ax[1])
ax[1].set_title('With SMOTE')

plt.show()
```

## 3. High Computational Cost
High computational cost is another major issue, as each boosting iteration carries a significant computational price in terms of both space and time due to the sequential creation of each new model/tree.

### 3.3 Parallel Implementation
The most effective solution is to use modern Gradient Boosting frameworks that have introduced parallelism techniques, even though the core algorithm is fundamentally sequential: **XGBoost** and **LightGBM**.
