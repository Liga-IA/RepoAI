# Modern Variants

The Computer Science community did not stop after Jerome Friedman introduced a different perspective on Boosting. Since then, several variants have been developed with the goal of making the algorithm's core idea even more robust.

## XGBoost:

XGBoost began as a research project led by Tianqi Chen at the **Distributed Machine Learning Community (DMLC)**. This variant, unlike classic Gradient Boosting which is developed on first-order gradients, uses a second-order Taylor expansion of the loss function through the Newton-Raphson method. The Newton-Raphson method finds the roots of a differentiable function **f**, which are the solutions to the equation **f(x) = 0**, by iteration, using a second-order Taylor expansion. This change provided a more precise and stable optimization.

XGBoost became quite famous in 2014 after being used in a solution for the **Higgs Boson Challenge**, which consisted of classifying particle collisions to identify the Higgs boson. Since then, it has been present in many Kaggle competitions.

Some of its advantages compared to classic Gradient Boosting are:

1.  **Speed:** It leverages memory, CPU, and parallelization optimizations, handling much larger datasets than classic Gradient Boosting.
2.  **Explicit Regularization:** It introduces L1 (Lasso) and L2 (Ridge) penalties on the tree leaves, helping to prevent overfitting.
3.  **Sparse Data:** It efficiently handles missing values or sparse data (variables with many zeros).
4.  **Flexibility:** It is suitable for many supervised learning models, such as Regression, Classification, and Ranking.

## LightGBM:

LightGBM, short for **Light Gradient-Boosting Machine**, is a variation of gradient boosting developed by Microsoft in 2016. Focused on speed and efficiency, this algorithm can handle large datasets. LightGBM emerged with the idea of solving the performance issue of existing GBDT algorithms. While more traditional models need to scan all the data for each feature in search of the split point, LightGBM created two new innovations to solve this: *Gradient-based One-Side Sampling* (GOSS) and *Exclusive Feature Bundling* (EFB).

-   **GOSS:** Instead of using all the data to train each tree, GOSS uses an intelligent sampling approach. The main idea is that instances (data rows) with small gradients are already well-trained, while those with large gradients (high errors) need more attention. Thus, GOSS keeps all instances with high gradients and selects only a random sample of the instances with low gradients. To avoid distorting the data distribution, it applies a larger weight to the sampled low-gradient instances.

-   **EFB:** In high-dimension datasets, it is common for several features to be "mutually exclusive," meaning they rarely have non-zero values at the same time. EFB intelligently exploits this by grouping (*bundling*) these features into a single "super-feature." To ensure no information is lost, EFB adds an offset to the values of each feature, causing them to occupy distinct value ranges within the new feature.

Some of its advantages are:

1.  **Extreme Speed:** Of the three algorithms, LightGBM is the fastest, being up to 20 times faster than conventional implementations. This is due to GOSS and EFB, but also to its *leaf-wise* tree growth strategy, which converges much faster.
2.  **Memory Efficiency:** By using a histogram-based approach and EFB, it consumes less memory than traditional models, making it ideal for large datasets.
3.  **Feature Reduction:** EFB significantly reduces the number of features to be processed, speeding up training and helping to better handle large datasets.
4.  **Categorical Data Handling:** LightGBM has a native and optimized implementation to handle categorical variables, eliminating the need for pre-processing.

## CatBoost:

CatBoost, which stands for "Categorical Boosting," was developed by the Russian technology company Yandex in 2017. The algorithm focuses on solving a problem present in other GBDT implementations, known as *target leakage*, which causes a *prediction shift*.

This problem manifests in two ways: in the handling of categorical variables and in the boosting process itself.

In the first case, a common solution is to replace a category with the average target value. However, by doing this, you use the target value itself to create a feature that will be used to predict that same value, causing data leakage. To solve this, CatBoost uses *Ordered Target Statistics*. For each data row, the numerical value of the category is calculated using only the average target of the rows that came before it in a random permutation of the database. This ensures that the target information of the row itself is never used to create its feature.

In the second case, when calculating the errors (gradients) to train the next tree, classic algorithms use a model that has already been trained on the same data, which generates biased gradients. To solve this, CatBoost applies the same logic with *Ordered Boosting*. To calculate the error of an instance, it uses a model that was trained only with the instances that preceded it in the permutation.

The main advantages of this model are:

1.  **Superior Quality with Categorical Data:** It is considered the best in its class for datasets with many categorical variables. Its *ordered target statistics* approach avoids overfitting and extracts the maximum information from these variables without requiring manual pre-processing.
2.  **Symmetric Trees:** The same splitting criterion is used across an entire level of the tree, which acts as a regularization and prevents the model from fitting too closely to noise in the data.
3.  **Ease of Use:** The algorithm requires less hyperparameter tuning compared to its competitors, automatically adjusting the learning rate during training in its default mode.
4.  **Prediction Speed:** Although its training is slower, the use of symmetric trees makes the process of making predictions extremely fast after the model has been trained.

## Tree Growth Strategy

One of the most fundamental differences between XGBoost, LightGBM, and CatBoost, which directly impacts the performance and final structure of the model, is the tree growth strategy.

The XGBoost tree grows in a balanced way, expanding level by level. As we see in the image, after the first split, all nodes at the same depth are split before the algorithm proceeds to the next layer. This more conservative approach helps control complexity and avoid excessively deep and specialized growth in just one branch of the tree.

LightGBM adopts a more aggressive, speed-focused strategy. Instead of expanding an entire level, it always splits the leaf that promises the greatest error reduction, regardless of its depth. The result, as shown in the image, is an asymmetric tree that deepens quickly in the most promising branches. This approach accelerates convergence but increases the risk of overfitting on smaller datasets.

CatBoost uses a unique structure called symmetric or *oblivious* trees. The rule is that all nodes at the same level must use the same splitting criterion. The image represents this by showing the same sequence of features in all branches (X1, then X2, then X3). This symmetric structure acts as a strong regularization, making the model more robust to overfitting and allowing for an extremely efficient prediction step.
