# Core Points of Gradient Boosting

## Introduction

Gradient Boosting is an ensemble model built through several **weak learners**—simple or basic predictive models—that attempt to correct the errors and residuals of previous ones. By the end of the training process, a **strong learner** is formed, creating a robust and reliable model for the task at hand.
These learners can range from simple neural networks, such as shallow MLPs, to low-depth decision trees. The latter are the most common choice for gradient boosting models because they are simple and applicable to almost any scenario.

## How it Works

In each iteration of the learners, the **pseudo-residuals** (the negative gradient of the loss function) are calculated. A new learner is then trained to approximate these residuals. Finally, all learners are combined to form the final model, either based on a predefined number of iterations or a target metric (early stopping). Furthermore, each learner is added to the model with a coefficient; the `learning_rate` parameter (**shrinkage**) controls this contribution, which also helps reduce overfitting when properly tuned.

## Advantages

Among the main advantages of gradient boosting are its high performance on **tabular data** and its relative interpretability. It is an excellent choice when dealing with tabular datasets or when the data volume does not justify the complexity of a large neural network. It is also effective in cases with limited data that is well-labeled and well-defined.

## Disadvantages

However, when dealing with more complex and unstructured data, gradient boosting may fall short in performance and results. It also requires careful **hyperparameter tuning** to achieve optimal results and avoid overfitting, although modern variants include built-in optimizations for these issues.

## Components

The main components and parameters used to build a gradient boosting model include:
*   **learning_rate**: Determines how much each learner affects the overall model.
*   **n_estimators**: The number of iterations (or trees).
*   **max_depth**: The maximum depth of the individual learners.
*   **subsample**: The fraction of samples used for training each learner.
