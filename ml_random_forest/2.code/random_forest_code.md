# Case Study: Classification with Random Forest

Este projeto aplica o algoritmo **Random Forest** para resolver dois problemas clássicos de classificação, demonstrando sua versatilidade em diferentes domínios de dados.

---

## 🇧🇷 Versão em Português

### Introdução aos Problemas
O algoritmo **Random Forest** é uma escolha poderosa para tarefas de classificação devido à sua robustez e capacidade de lidar com dados complexos e multidimensionais. Utilizaremos este modelo para os seguintes desafios:

1.  **Dataset de Vinhos (Wine Dataset):**
    *   **Objetivo:** Classificar a origem (cultivar) de vinhos com base em suas características químicas.
    *   **Atributos:** Possui 13 variáveis contínuas, incluindo teor alcoólico, ácido málico, cinzas e intensidade de cor.
    *   **Desafio:** Identificar padrões em dados químicos altamente correlacionados para distinguir entre três classes de vinhos.

2.  **Dataset de Dígitos (Digits Dataset):**
    *   **Objetivo:** Reconhecer dígitos manuscritos (de 0 a 9).
    *   **Atributos:** Imagens de $8 \times 8$ pixels, onde cada atributo representa a intensidade de cinza de um pixel (64 dimensões).
    *   **Desafio:** Classificar corretamente o dígito a partir de uma representação numérica simplificada de imagem, lidando com a alta dimensionalidade dos dados.

## 🇺🇸 English Version

### Introduction to the Problems
The **Random Forest** algorithm is a powerful choice for classification tasks due to its robustness and ability to handle complex, high-dimensional data. We will use this model for the following challenges:

1.  **Wine Dataset:**
    *   **Goal:** Classify the origin (cultivar) of wines based on their chemical profiles.
    *   **Attributes:** Consists of 13 continuous variables, including alcohol content, malic acid, ash, and color intensity.
    *   **Challenge:** Identify patterns in highly correlated chemical data to distinguish between three distinct wine classes.

2.  **Digits Dataset:**
    *   **Goal:** Recognize handwritten digits (from 0 to 9).
    *   **Attributes:** $8 \times 8$ pixel images, where each attribute represents the grayscale intensity of a pixel (64 dimensions).
    *   **Challenge:** Correcty classify the digit from a simplified numerical image representation, handling the high-dimensional nature of the feature set.
